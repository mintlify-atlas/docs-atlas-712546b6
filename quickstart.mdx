## Prerequisites

Before you begin, make sure the following tools are installed:

- **Docker** (v20.10 or later)
- **Docker Compose** (v2 or later â€” included with Docker Desktop)
- **Git**

## Setup

<Steps>
  <Step title="Clone the repository">
    Clone the Xoxemons repository and enter the project root:

    ```bash
    git clone https://github.com/MarcFrancesCharles/XOXEMONS.git
    cd XOXEMONS
    ```
  </Step>

  <Step title="Configure the backend environment">
    Copy the example environment file for the Laravel backend:

    ```bash
    cp backend/.env.example backend/.env
    ```

    <Note>
      The default values in `.env.example` already match the database credentials defined in `docker-compose.yml` (`DB_HOST=db`, `DB_DATABASE=backend`, `DB_USERNAME=root`, `DB_PASSWORD=root`). You do not need to change anything for local development.
    </Note>
  </Step>

  <Step title="Build and start the containers">
    From the project root, build all images and start the services in detached mode:

    ```bash
    docker-compose up -d --build
    ```

    This command builds the Angular development server and the PHP/Apache backend image, then starts four containers: `db`, `adminer`, `backend`, and `frontend`. The first build may take a few minutes.
  </Step>

  <Step title="Run migrations and seed the database">
    Enter the backend container and initialise the database:

    ```bash
    docker-compose exec backend bash
    ```

    Inside the container, run the following commands one at a time:

    ```bash
    composer install
    php artisan key:generate
    php artisan migrate:fresh --seed
    ```

    Type `exit` when done to leave the container shell.

    The seeder creates the admin account, five test players, the full Xoxemon catalogue, all items, and the default game settings.
  </Step>

  <Step title="Open the application">
    The application is now running. Open these URLs in your browser:

    | Service | URL | Description |
    |---------|-----|-------------|
    | Frontend | `http://localhost:4200` | Angular game client |
    | Backend API | `http://localhost:8000/api` | Laravel REST API |
    | Adminer | `http://localhost:8080` | Database management UI |
  </Step>
</Steps>

## Troubleshooting

<Tip>
  If `docker-compose up` fails with a port conflict, another process is already using port `3306`, `4200`, `8000`, or `8080`. Stop the conflicting service or change the host-side port mapping in `docker-compose.yml` (the number on the left of the colon).

  If the frontend shows a blank screen or network errors after the containers start, wait 30â€“60 seconds for the Angular dev server to finish compiling, then refresh the page.

  If `php artisan migrate:fresh --seed` returns a database connection error, the MySQL container may still be starting. Run `docker-compose ps` to confirm the `db` container is healthy, then retry.
</Tip>
