---
title: "GET /api/me — Fetch the authenticated user profile"
description: "Returns the full profile for the currently authenticated user, including coins, role, last daily reward timestamp, and custom_id. Requires a valid JWT."
---

The `/me` endpoint returns the complete profile record for the player who owns the current JWT token. It is the primary way to check the authenticated user's state — coins, role, and whether a daily reward is available. This page also covers the related profile management endpoints: update profile and delete account.

**`GET /api/me`** — Requires `Authorization: Bearer <token>`.

## Response

<ResponseField name="id" type="integer">Unique database ID.</ResponseField>
<ResponseField name="custom_id" type="string">Player's unique identifier, e.g. `Marc#4721`.</ResponseField>
<ResponseField name="name" type="string">First name.</ResponseField>
<ResponseField name="surnames" type="string">Surname(s).</ResponseField>
<ResponseField name="email" type="string">Email address.</ResponseField>
<ResponseField name="role" type="string">`player` or `robot`.</ResponseField>
<ResponseField name="coins" type="integer">Current coin balance.</ResponseField>
<ResponseField name="last_daily_reward" type="string|null">ISO 8601 timestamp of last daily claim, or `null` if never claimed.</ResponseField>
<ResponseField name="created_at" type="string">Account creation timestamp.</ResponseField>

## Example

```bash
curl http://localhost:8000/api/me \
  -H "Authorization: Bearer eyJ0eXAiOiJKV1Qi..."
```

```json
{
  "id": 5,
  "custom_id": "Marc#4721",
  "name": "Marc",
  "surnames": "Frances",
  "email": "marc@example.com",
  "role": "player",
  "coins": 0,
  "last_daily_reward": "2024-10-01T08:00:00.000000Z",
  "created_at": "2024-09-15T10:00:00.000000Z"
}
```

## Related endpoints

### Update profile — `PATCH /api/user/profile`

Update one or more profile fields. All fields are optional; only send the ones you want to change.

<ParamField body="name" type="string">New first name.</ParamField>
<ParamField body="surnames" type="string">New surnames.</ParamField>
<ParamField body="email" type="string">New email — must be unique across all accounts.</ParamField>
<ParamField body="password" type="string">New password (min 6 chars). Must include `password_confirmation`.</ParamField>
<ParamField body="password_confirmation" type="string">Required when changing password.</ParamField>

Returns `200 OK` with a `message` and updated `user` object. Returns `422` if validation fails.

### Claim daily reward — `POST /api/user/daily-reward`

Awards one random Small Xoxemon and 10 sweets once per calendar day. No request body required.

Returns `200 OK` with a confirmation message, or `400` if already claimed today.

### Delete account — `DELETE /api/user/account`

Permanently deletes the authenticated account and all associated Xoxemons, items, friendships, and messages (via database cascade). The JWT is invalidated before deletion.

<Warning>
  Account deletion is irreversible. All Xoxemons and inventory items are lost.
</Warning>
